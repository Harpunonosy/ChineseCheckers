These diagrams provide a visual representation of the project\textquotesingle{}s architecture and design patterns used.\hypertarget{index_autotoc_md10}{}\doxysection{\texorpdfstring{Design Patterns}{Design Patterns}}\label{index_autotoc_md10}
The project utilizes several design patterns, including\+:


\begin{DoxyEnumerate}
\item {\bfseries{Singleton}}\+: Ensures that there is only one instance of the server managing all connections and games.
\item {\bfseries{Abstract Factory}}\+: Allows for the creation of different game variants (standard and custom boards and rules) without changing the main game logic.
\item {\bfseries{Factory Method}}\+: Responsible for creating appropriate instances of boards and game rules.
\item {\bfseries{Observer}}\+: Notifies players about moves, allowing them to see the effects of moves made by other players.
\item {\bfseries{Strategy}}\+: Allows for dynamic assignment of different sets of game rules.
\item {\bfseries{Facade}}\+: Simplifies client operations by providing a unified interface to manage the connection, input, and output.
\item {\bfseries{State}}\+: Manages the different states of the game (waiting for players, player\textquotesingle{}s turn, game over) and changes the behavior of the game based on its state.
\end{DoxyEnumerate}\hypertarget{index_autotoc_md11}{}\doxysection{\texorpdfstring{License}{License}}\label{index_autotoc_md11}
